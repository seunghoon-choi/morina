1. 서비스 정의 및 대상
────────────────────────────────────────────────────────────────
서비스명: ByeTax (바이택스)
캐릭터: 세비
주요 대상: 도소매, 서비스, 음식점(간이 포함), 운송, 인적용역(프리랜서),
          교육, 전자상거래 등 간편장부 및 자기조정 대상 사업자.
핵심 가치: 일상적인 경비 관리부터 원천세·부가세·종합소득세 신고까지,
          세무사 없이 스스로 마칠 수 있도록 돕는
          '밀착형 세무 내비게이션'.


2. 주요 기능 요구사항
────────────────────────────────────────────────────────────────

[A] 상시 세무 관리 및 진단 엔진
  - 데이터 수집/정리:
    시스템이 자동 수집하는 방식이 아닌, 사용자가 홈택스에서 다운로드한
    '매입세금계산서 목록조회 자료'를 직접 바이택스에 업로드하는 구조로 구현.
    업로드된 파일 내의 공급가액, 부가세, 상호명 등을 추출하여
    경비 리스트로 자동 변환.
  - 세금계산서 발행되지 않은 경비(기타 영수증, 종이세금계산서로 발행된
    임대료, 이체 내역 등)의 즉시 기록 및 데이터화.
  - 인건비, 접대비, 감가상각비, 대출이자 등의 경비를 직접 입력해
    납부할 세금에 가까운 금액 산출.
  - 실시간 세금 예측: 누적된 경비와 업종별 경비율을 바탕으로
    예상 소득세 및 부가세를 수시로 산출.
  - 절세 가이드: 누락된 자료를 체크하고, 현재 지출 구조에 맞는
    절세 전략을 푸시 알림으로 가이드.

[B] 3대 핵심 세금 신고 절차 지원 (Navigation)
  - 대상 세목: 원천세, 부가세, 종합소득세.
  - 하이브리드 신고 UI:
    · 모바일(바이택스): PC로 홈택스 신고를 진행할 때,
      모바일은 옆에서 '입력값 제공 및 단계별 가이드 화면' 역할을 수행함.
      "지금 PC 화면의 A 칸에 이 숫자를 넣으세요"라는 지침을
      최적화하여 보여줌.
    · PC(홈택스): 실제 신고 수행.
      (신고 실행 화면: 홈택스 창을 옆에 띄워두고 작업할 수 있도록
       최적화된 레이아웃 제공.)
  - 관리자 전용 페이지 (Admin):
    · PC 버전에서만 접근 가능.
    · 전체 회원 관리, 결제 현황, 시스템 진단 로직 설정 및 공지사항 관리.
      사용자 상담요청 모니터링 기능 포함.
  - 매칭 로직: "홈택스 [A] 칸에 바이택스의 [B] 항목 숫자를 입력하세요"와
    같이 화면을 1:1로 매칭하여 안내하는 절차 지원 시스템 구축.

[C] 종합소득세 최종 취합 및 신고
  - 일 년간 바이택스에 정리된 모든 경비와 신고 내역을 자동 취합.
  - 홈택스 종합소득세 신고서 서식에 맞게 데이터를 재구성하여
    사용자가 직접 신고서를 완성하도록 지원.


3. 수익 모델 및 비즈니스 로직
────────────────────────────────────────────────────────────────
- 멤버십 서비스 (월 14,900원):
  · 상시 밀착 관리 및 정밀 진단 서비스 제공.
  · 신고 절차 가이드 및 데이터 무제한 보관.
- 세무사 파트너십 연동:
  · 상담원으로 어렵고 자가 신고가 어려운 고난도 케이스 발생 시
    파트너 세무사와 즉시 연결.
  · 연결 시 발생하는 수수료 기반의 수익 모델 연동.


4. 시스템 지침 (System Prompt)
────────────────────────────────────────────────────────────────
- 모든 가이드는 '사용자가 홈택스를 열어놓고 따라 하는 상황'을 전제로
  텍스트와 시각 자료를 생성할 것.
- 복잡한 세무 용어보다는 소상공인이 이해하기 쉬운 실무 용어로 변환하여
  제공할 것.
- 데이터의 최종 목적지는 '종합소득세 신고서'임을 인지하고, 모든 경비
  데이터를 소득세 서식에 맞춰 구조화할 것.


5. 인증 및 커뮤니케이션 (API 연동)
────────────────────────────────────────────────────────────────
- 간편 로그인: 카카오톡 로그인 기능을 기본 인증 수단으로 적용.
- 메시징 서비스: 팝빌(Popbill) API를 연동하여 시스템 내 알림톡,
  문자(LMS/SMS) 발송 기능 구현.


6. 인원(인건비) 관리 특화 기능
────────────────────────────────────────────────────────────────
신분증 수집 자동화 워크플로우:
  1. 사용자(사장님)가 앱 내에서 '신분증 요청' 버튼 클릭.
  2. 시스템이 팝빌 API를 통해 직원에게 카카오 알림톡/문자로
     신분증 첨부 링크 전송.
  3. 직원이 해당 링크를 통해 직접 신분증 사진을 첨부하여 제출.
  4. 제출된 파일은 바이택스 내 해당 직원의 인적자원 데이터로 자동 저장.


[공통 개발 요청]
모바일과 PC는 100% 동일한 기능을 갖춘 반응형 구조로 설계하되,
신고 시 모바일은 가이드 전용 UI를 제공하고,
관리자 페이지는 PC 환경에서만 활성화되도록 개발 요청.

■ 총평
────────────────────────────────────────────────────────────────
전체적으로 포지셔닝이 상당히 영리한 기획.
다만 4가지 지점을 한 번 더 다듬어야 실행력 있는 스펙이 됨:
  1) 타겟 좁히기
  2) 가격 정당화 스토리
  3) 법무 리스크 (신분증)
  4) 모바일-PC 연동 UX 현실성


■ 강점 — 이 부분들이 좋았음
────────────────────────────────────────────────────────────────

1) "자동화"가 아니라 "내비게이션" 포지셔닝
   - 이게 핵심 차별점.
   - 삼쩜삼·자비스·기장부는 전부 "대신 해드립니다" 쪽인데,
     바이택스는 "홈택스는 직접, 우리는 길 안내"
   - 장점:
     · 세무사법(제20조) 회피 — 대행이 아니라 도구 제공이라 법적 리스크 낮음
     · 홈택스 스크래핑/API 의존 제거 — UI 변경에 치명적이지 않음
     · 책임 분산 — 최종 신고 책임이 사용자에게 있음

2) PDF 업로드 방식
   - 이미 MVP에서 구현된 구조가 이 기획의 A 모듈과 정확히 일치.
   - 기존 자산 그대로 활용 가능.

3) 팝빌 + 신분증 수집 워크플로우
   - 소상공인이 진짜로 겪는 pain point.
   - 직원 4대보험 신고 시 신분증을 카톡/문자로 산발적으로 받는 문제를
     링크 한 번으로 해결 — 강력한 기능.


■ 짚어보고 싶은 지점들
────────────────────────────────────────────────────────────────

1) 가격(₩14,900/월)의 정당화가 약함
   사용자 입장 비교:
     - 홈택스 모두채움 신고     → 무료 (단순 케이스)
     - 삼쩜삼                   → 신고 시점 환급액 일부만 (상시비용 없음)
     - 세무사 기장              → 월 ₩10만대부터 (완전 대행)
     - 바이택스                 → 연 ₩17.8만 + 결국 내가 신고

   "연 18만원 내고 내가 직접 한다"가 매력적이려면
   사용자가 연간 세무사비 50만원+ 아낀다는 체감이 있어야 함.
   그걸 만들려면:
     · 실시간 예상세액 정확도 (±5% 이내)
     · "이거 놓치면 세금 X원 더 내요" 같은 구체적 절세 액션 가이드
     · 신고 당일 진짜로 1시간 안에 끝나는 경험

   대안: "무료 진단 → 신고 시즌만 유료" 모델도 고민해볼 가치 있음.

2) 타겟이 너무 넓음
   도소매/서비스/음식점/운송/프리랜서/교육/전자상거래 7개 업종은
   세무 특성이 전부 다름:
     - 음식점        → 의제매입세액, 신용카드 매출 비율
     - 프리랜서      → 원천징수 차감, 사업소득 vs 기타소득 구분
     - 전자상거래    → 스마트스토어/쿠팡/11번가 플랫폼별 매출신고,
                      간이→일반 전환 타이밍
     - 인적용역      → 강사료/자문료 구분, 3.3% vs 8.8%

   MVP 단계에서는 1~2개 업종에 집중하고
   "이 업종은 바이택스가 최고"를 만든 뒤 확장하는 게 실패 확률 낮음.
   추천: 프리랜서/인적용역 (페르소나 명확, 상시관리 니즈 있음).

3) "신분증 수집" — 법무 체크 필수
   매력적이지만 개인정보보호법 리스크가 큼:
     · 신분증 = 주민등록번호 포함 고유식별정보 (제24조)
     · 수집·이용·보관에 별도 동의 필요
     · 암호화 저장 의무, 최소 수집, 목적 달성 후 즉시 파기
     · 유출 시 과징금 최대 전체 매출의 3%

   개발 전 개인정보보호 전문가 자문 필수.
   설계 단계에서 "이미지 자체 저장 안 하고 암호화 전달만" 구조
   (또는 HSM 수준 암호화) 확정 필요.

4) "모바일 가이드 + PC 홈택스" UX 현실성 테스트 필요
   기획상 좋지만 실제 시나리오 문제:
     · 한 손에 폰, 한 손에 마우스 잡고 화면 전환하며 숫자 입력
     · 홈택스 연 2회 UI 개편 → "A칸" 위치 바뀌면 매핑 재작업
     · 스크린샷 가이드라면 버전관리 콘텐츠 지옥

   대안:
     · PC 홈택스 + PC 바이택스 분할 화면 최적화를 1차로
     · 모바일 가이드는 나중에 추가
     · 또는 크롬 확장프로그램으로 홈택스 옆에 플로팅 가이드
       (유지보수 훨씬 쉬움)

5) 현재 MVP와의 관계를 정리해야 함
   지금 만들어둔 바이택스 MVP는 [C] 종합소득세 취합·계산 쪽이고,
   이 기획은 [A]+[B]+[C] 전부 포괄하는 훨씬 큰 그림.

   옵션 1 (확장): 현 MVP 유지 → [A] 매입세금계산서 업로드 추가 →
                [B] 가이드 UI 추가
   옵션 2 (리셋): DB 스키마/구조 전체 아키텍처 재설계, 처음부터

   추천: 옵션 1.
   이유: 현 MVP가 "종소세 신고 시즌(5월) 도우미"로 이미 시장 진입
        가능한 상태. 유저 피드백 받으며 [A]/[B]로 확장하는 게
        학습 효율 높음.


■ 확인이 필요한 질문들
────────────────────────────────────────────────────────────────
1. 이 기획을 누구에게 먼저 팔 건가?
   — 타겟 1순위 업종/페르소나 1개만 고른다면?
2. "세비" 캐릭터의 역할은?
   — 단순 마스코트인지, AI 챗봇 페르소나인지?
3. 개발 순서 우선순위 — [A]/[B]/[C] 중 어느 것부터?
4. 경쟁사 조사하신 게 있나?
   — 삼쩜삼·자비스·기장부와 1문장 차별점 정의?
5. 현 MVP의 PDF 파싱 자산을 [A] 모듈로 연결하는 게 맞는지 확인
