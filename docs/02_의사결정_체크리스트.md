## 1️⃣ 세무건강도 산정 공식

### 배경
화면 스케치 페이지 1의 "72점" 게이지. 안전(80-100) / 주의(50-79) / 위험(0-49) 구간만 있고 **공식 없음**. 숫자 산정 로직 없이는 사용자 신뢰도 구축 불가.

### 옵션

#### 옵션 A: 5지표 가중평균 (★ 권장)
```
세무건강도 = Σ (지표 × 가중치)

지표 1. 신고 기한 준수도     (25%)  — 가산세 리스크
지표 2. 경비율 적정성       (25%)  — 업종 평균 대비
지표 3. 증빙 완비율         (20%)  — 누락 자료 비율
지표 4. 전년 대비 변동성     (15%)  — 비정상 증감
지표 5. 현금지출 비율        (15%)  — 카드/계좌 대비
```
- **장점**: 다면 평가, 설명 가능
- **단점**: 각 지표별 세부 공식 필요
- **산정 예시**:
  - 신규창업자: 지표 1·4 산정 불가 → 가중치 재분배
  - 프리랜서: 지표 3·5 비중 ↑

#### 옵션 B: 3지표 단순 가중평균
```
세무건강도 = (소득률 30%) + (경비율 40%) + (증빙완비율 30%)
```
- **장점**: 단순, 빠른 구현
- **단점**: 다양한 리스크 반영 불가

#### 옵션 C: 룰 기반 페널티 감점
```
100점에서 시작 → 문제 건수에 따라 감점
 - 신고 지연 1회: -20점
 - 경비율 업종평균 이상: -15점
 - 증빙 누락 10건당: -5점
 - 현금 지출 30% 초과: -10점
```
- **장점**: 사용자가 개선 포인트를 직관적으로 파악
- **단점**: 감점 체계 조율 어려움

### 결정 포인트
☐ 옵션 A (5지표 가중평균) ← 권장
☐ 옵션 B (3지표 단순평균)
☐ 옵션 C (룰 기반 감점)
☐ 기타: ___________________

### 권장 이유
- 옵션 A는 **설명 가능성**이 뛰어남. 사용자가 점수 클릭 시 5지표 상세가 펼쳐지는 UX 가능
- "왜 72점인지" 응답 가능해야 신뢰 구축됨
- 업종별 가중치 재분배로 유연성 확보

## 2️⃣ 부가세·종소세 가이드 깊이

### 배경
원천세 가이드는 **5페이지(p.14~18)** 할애하여 매우 상세. 하지만 부가세·종소세는 명확한 가이드 없음. 비즈니스 모델 "[B] 3대 핵심 세금 신고 절차 지원"의 완성도에 직결.

### 옵션

#### 옵션 A: 3대 세목 동일 수준 (★ 권장)
- 원천세 가이드 구조를 **템플릿화**하여 부가세·종소세에 복제
- 각 세목마다 5~7페이지 분량의 홈택스 입력 가이드
- **장점**: 서비스 완성도, 사용자 체감 가치 극대화
- **단점**: 개발 공수 ~6주 추가

#### 옵션 B: 세목별 차등
| 세목 | 깊이 |
|---|---|
| 종소세 | ★★★★★ (5~7 페이지 가이드) |
| 부가세 | ★★★★ (3~4 페이지) |
| 원천세 | ★★★ (이미 완성됨) |

- 근거: 종소세가 가장 복잡, 연 1회이지만 임팩트 큼
- **장점**: 중요도에 맞춘 투자
- **단점**: 부가세 사용자 경험 상대적 약함

#### 옵션 C: v1은 원천세만, 부가세·종소세는 v2
- MVP는 원천세만 완성 (음식점 타겟 적합)
- 부가세는 **외부 링크 + 간단 체크리스트**
- 종소세는 **기존 MVP 계산기 활용**
- **장점**: 빠른 출시 가능 (3개월)
- **단점**: 3대 세금 통합 가치 제안 실종

### 결정 포인트
☐ 옵션 A (3대 세목 동일)
☐ 옵션 B (차등, 종소세 강화) ← 음식점 타겟 시 권장
☐ 옵션 C (원천세 우선 v1)

### 세목별 개발 예상 공수 (참고)
| 세목 | 엔진 개발 | UI 가이드 | 합계 |
|---|---|---|---|
| 원천세 (완성) | 0 | 0 | - |
| 부가세 | 3주 | 2주 | 5주 |
| 종소세 | 기존 MVP 활용 | 3주 | 3주 |

## 3️⃣ 알림 채널

### 배경
세무 일정 D-day 알림이 **핵심 리텐션 도구**. 어떤 채널로 알림을 보낼지 결정 필요.

### 옵션

#### 옵션 A: 앱 푸시 + 팝빌 알림톡 조합 (★ 권장)
```
통상 알림: 앱 푸시 (무료, 즉시성 좋음)
중요 알림: 카톡 알림톡 (건당 ~10원, 도달률 95%+)
긴급 알림: 알림톡 + SMS 이중 (건당 ~20원, 도달률 99%)
```
- **장점**: 비용 최적화, 중요도별 차등
- **단점**: 로직 복잡

**알림 중요도 분류**:
| 등급 | 예시 | 채널 |
|---|---|---|
| 긴급 🔥 | 신고 당일, 가산세 임박 | 앱푸시 + 알림톡 + SMS |
| 중요 ⚠️ | 신고 D-7, D-3 | 앱푸시 + 알림톡 |
| 일반 ✉️ | 월간 리포트, 팁 | 앱푸시만 |

#### 옵션 B: 카톡 알림톡 중심
- 모든 알림을 카톡 알림톡으로 통일
- **장점**: 사용자 도달률 높음, 앱 미사용자도 수신 가능
- **단점**: 건당 비용 발생 (1,000명 × 월 5건 = 50,000건 × 10원 = 월 50만원)

#### 옵션 C: 앱 푸시만
- FCM 활용, 무료
- **장점**: 비용 제로
- **단점**: 앱 미설치/알림 차단 시 도달 안 됨

### 결정 포인트
☐ 옵션 A (조합) ← 권장
☐ 옵션 B (알림톡 중심)
☐ 옵션 C (앱 푸시만)

### 팝빌 비용 (참고)
- 알림톡: **9~12원/건**
- SMS: **18~20원/건**
- LMS (장문): **30~40원/건**

### 월 비용 추정 (구독자 1,000명 기준)
| 옵션 | 월 비용 |
|---|---|
| A | 약 10~30만원 |
| B | 약 40~60만원 |
| C | 0원 (도달률 낮음) |

## 4️⃣ 보관함 용량/기간 정책

### 배경
보관함은 **가장 강력한 Lock-in 장치**. 사용자가 "여기서 빠져나가면 데이터 잃는다"는 심리를 형성. 하지만 정책 설계 미스 시 스토리지 원가 폭발 또는 사용자 불만 발생.

### 옵션

#### 옵션 A: 용량 무제한 + 기간 차등 (★ 권장)
```
무료: 1개월 보관 (이후 자동 삭제)
구독: 5년 보관 (법정 의무 기간)
구독 해지: 즉시 열람 불가, 복구 30일 유예
```
- **장점**: 사용자 불편 최소화, 세법 준수 명확
- **단점**: 스토리지 원가 증가 (하지만 문서 중심이라 소액)

**예상 원가**:
- 평균 유저당 5GB 저장 × ₩30/GB/월 = ₩150/월
- 1,000명 × ₩150 = 월 ₩150,000 (감당 가능)

#### 옵션 B: 용량 차등 + 기간 무제한
```
무료: 1GB (부족하면 이용 불가)
구독: 30GB
프로: 100GB
```
- **장점**: 매출 upsell 기회
- **단점**: 구글드라이브와 비교 당함 (앞서 논의 결론)

#### 옵션 C: 완전 무제한 (구독자)
```
무료: 1개월 × 1GB
구독: 용량·기간 무제한
```
- **장점**: 최고의 UX
- **단점**: 장기적 스토리지 리스크 (유저당 50GB 초과자 발생)

### 결정 포인트
☐ 옵션 A (용량 무제한 + 기간 차등) ← 권장
☐ 옵션 B (용량 차등 + 기간 무제한)
☐ 옵션 C (완전 무제한)

### 법적 참고
- **국세기본법**: 장부·증빙 **5년 보관 의무**
- **전자세금계산서**: **5년 보관**
- **개인정보**: 퇴사일로부터 **3년 후 파기**
- → **5년이 표준 법정 기간**

## 5️⃣ 세무사 매칭 로직

### 배경
세무사 파트너십은 **부가 수익 모델**. 사용자가 "검토 예약" 버튼 클릭 시 어떤 세무사에게 연결할지 결정 필요.

### 옵션

#### 옵션 A: 지역+업종 기반 수동 매칭 (★ v1 권장)
```
1. 사용자 사업장 지역 확인
2. 파트너 세무사 중 해당 지역 담당자 필터
3. 업종 전문성 매칭 (음식점·전자상거래 등)
4. 운영자가 최종 매칭 (수동)
5. 세무사 응답 → 사용자 알림
```
- **장점**: MVP 단계 적합, 품질 관리 가능
- **단점**: 운영자 개입 필요 (스케일업 어려움)

**초기 파트너 세무사 수**: 5~10명으로 시작

#### 옵션 B: 자동 라운드 로빈
```
1. 파트너 세무사 풀에서 순차 매칭
2. 세무사 수용 거부 시 다음 매칭
3. 24시간 내 매칭 실패 시 운영자 개입
```
- **장점**: 확장성 좋음
- **단점**: 품질/전문성 매칭 약함

#### 옵션 C: 사용자 선택형
```
1. 사용자에게 파트너 세무사 리스트 제공
2. 경력·업종·지역·리뷰 기반 선택
3. 사용자가 직접 선택
```
- **장점**: 투명성 높음
- **단점**: 사용자 판단 부담, 리뷰 시스템 구축 필요

### 결정 포인트
☐ 옵션 A (수동 매칭, v1) ← 권장
☐ 옵션 B (자동 라운드 로빈)
☐ 옵션 C (사용자 선택형)

### 수익 모델
- **사용자 → 바이택스**: 상담 예약 무료 (랜딩 방식)
- **세무사 → 바이택스**: 매칭 수수료 (수임료의 10~20%)
- 또는 **구독형**: 세무사가 월 파트너십 비용 지불
